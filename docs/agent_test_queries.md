# Тестові кейси-запити для відладки агента-помічника

Документ містить набір запитів і очікувану поведінку для перевірки логіки обробки запитів, формату відповіді та відсутності небажаного експорту даних.

---

## 1. Аналітика лише текстом (без експорту)

**Що перевіряємо:** Запит на аналітику/підсумок не повинен призводити до вивантаження файлів. Відповідь — текст з числовими даними.

| Запит | Очікування |
|-------|------------|
| Дай аналітику зміни цін за кв. м. в Києві та області за останній тиждень по дням | **0 файлів.** Текстова відповідь з числами (дати, значення, одиниці). Агент викликає execute_analytics (ProZorro/OLX), не save_query_to_temp_collection + export. |
| Покажи підсумок по середній ціні за м² за останній тиждень по регіонах | **0 файлів.** Текст з агрегованими даними. |
| Скільки оголошень OLX за останній тиждень у Києві? | **0 файлів.** Коротка відповідь з числом (наприклад «Знайдено N оголошень»). |

**Критично:** Після відповіді на такий запит агент **не повинен** сам ініціювати експорт «на всяк випадок» (як у кейсі з вивантаженням 391 рядка без явного запиту на файл).

---

## 2. Звіт / експорт за період (очікуємо файли)

**Що перевіряємо:** Інтерпретатор визначає intent і response_format; мультиагентний пайплайн або LangChain виконують save_query + export.

| Запит | Очікування |
|-------|------------|
| Звіт за добу | **Файли** (prozorro + olx за останню добу). Текст підкріплений числами (кількість записів у файлах). |
| Сформуй звіт за добу по нежитловій нерухомості | **Файли** лише з оголошеннями/аукціонами по нежитловій нерухомості за останню добу. Інтерпретатор виставляє `intent=report_last_day`, `period_days=1`, `filters.property_type=\"Нерухомість\"`, планувальник додає фільтр за типом об’єкта через приєднання `llm_cache`. |
| Експорт за тиждень | **Файли.** Текст з підсумком (скільки файлів, скільки записів). |
| Виведи оголошення за останню добу в Excel | **Файли.** |

---

## 3. Запити без вибірки даних

**Що перевіряємо:** Немає викликів вибірки/експорту; відповідь лише текстом.

| Запит | Очікування |
|-------|------------|
| Що таке метрика average_price_per_m2? | **0 файлів.** Пояснення метрики, без execute_query/export. |
| Які колекції доступні? | **0 файлів.** Текст (get_allowed_collections або схема). |
| Допоможи сформулювати запит для аналітики | **0 файлів.** Консультація, без виконання вибірки. |

---

## 4. Відповідь із числовими даними при вибірці

**Що перевіряємо:** Навіть коротка текстова відповідь підкріплена числами (логіка з системного промпту та AssistantAgent).

| Запит | Очікування |
|-------|------------|
| Скільки аукціонів за сьогодні? | Текст **містить число** (наприклад «Знайдено N аукціонів» або «0»). |
| Підсумок по OLX за тиждень | Текст **містить числа** (кількість, можливо середня ціна тощо). |

---

## 5. Інтерпретатор (intent, needs_data, response_format)

**Що перевіряємо:** Для кожного запиту перевірити, що `interpret_user_query` повертає очікувані поля.

| Запит | Очікувані поля (орієнтовно) |
|-------|-----------------------------|
| Звіт за добу | intent=report_last_day, needs_data=True, response_format=files |
| Дай аналітику по днях | intent=query, needs_data=True, response_format=general_text або files (за ключовими словами) |
| Що таке метрика X? | intent=query, needs_data=False, response_format=general_text |
| Експорт за тиждень в Excel | intent=report_last_week або export_data, needs_data=True, response_format=files |

---

## 6. Регресія: не експортувати без явного запиту

**Сценарій:** Користувач задає один запит (наприклад аналітику по днях), отримує текстову відповідь. **Без додаткового повідомлення** агент не повинен виконувати save_query_to_temp_collection + export_from_temp_collection.

**Перевірка:** Після запиту типу «Дай аналітику зміни цін за кв. м. в Києві та області за останній тиждень по дням» у логах не повинно бути викликів export_from_temp_collection або save_query_to_temp_collection для «рандомної» вибірки (наприклад 391 запис OLX без уточнення періоду/фільтрів у запиті користувача).

---

## Як використовувати

- **Ручна відладка:** Копіювати запити в чат (Telegram / Mini App) і перевіряти наявність/відсутність файлів та зміст тексту.
- **Скрипт:** `scripts/run_agent_test_queries.py` — прогон підмножини кейсів через MultiAgentService з перевіркою очікувань (див. опис у скрипті).
- **Тестування агентом (звіт по БД):** У меню **Адміністрування** → **🧪 Тестування агента**. Тест-агент генерує 5 кейсів (через LLM або резервний набір), прогоняє їх через LLM-помічника, перевіряє результати прямим доступом до БД (усі колекції) та надсилає детальний звіт з «ходом думок» і підсумком по кожному кейсу. Сервіс: `business/services/agent_test_runner_service.py`.
