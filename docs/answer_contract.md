# Контракт відповіді (Answer Contract)

Єдиний формат відповіді від бекенду клієнтам (Telegram-бот, Mini App). Формується **Answer Composer** після виконання запиту (аналитичний пайплайн, звіт, експорт).

## Поля контракту

| Поле | Тип | Опис |
|------|-----|------|
| `type` | string | Тип відповіді: `list`, `table`, `chart`, `file`, `text` |
| `title` | string | Заголовок результату |
| `items` | array | Список елементів для відображення (кожен: `text`, опційно `index`, інші поля) |
| `attachments` | array | Вкладення: `{ "artifact_id" }` або `{ "file_base64", "filename" }` |
| `summary` | string | Текстова відповідь для користувача (основне повідомлення) |

## Типи відповіді

- **list** — короткий список (наприклад топ-10), до ~10–20 елементів.
- **table** — табличні дані, можна відобразити як markdown або список.
- **chart** — дані для графіка (тренд, розподіл); клієнт може візуалізувати.
- **file** — великий результат; основний контент у файлі, `summary` — короткий підсумок.
- **text** — проста текстова відповідь (наприклад «результатів не знайдено»).

## Використання на клієнті

- **Telegram**: відправити `summary` як текст повідомлення; якщо `attachments` не порожній — відправити файли окремо (`send_document`).
- **Mini App**: відобразити `title`, `items` у списку/таблиці; `summary` — у блоці підсумку; файли — посилання або кнопка завантаження.

## Джерело

- Сервіс: `business/services/answer_composer_service.py`
- Вхід: результат виконання (дані, тип запиту, кількість рядків, наявність файлу) + контекст клієнта (канал, обмеження довжини).
