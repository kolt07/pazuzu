# Пайплайн отримання та фільтрації даних ProZorro

## Загальна схема

```
main.py::fetch_real_estate_auctions()
    ↓
prozorro_service.py::fetch_and_save_real_estate_auctions()
    ↓
prozorro_service.py::get_real_estate_auctions()
    ↓
prozorro_service.py::_get_auctions_internal()
    ├─→ _get_auctions_by_date_modified()  [ЕТАП 1: Отримання даних]
    ├─→ _get_auctions_by_date_created()   [ЕТАП 1: Отримання даних]
    └─→ Об'єднання та дедуплікація        [ЕТАП 2: Об'єднання]
    ↓
    Фільтрація через _should_include_auction()  [ЕТАП 3: Фільтрація]
    ↓
prozorro_service.py::save_auctions_to_csv()  [ЕТАП 4: Збереження]
```

---

## ЕТАП 1: Отримання даних з API

### 1.1. Отримання за датою модифікації (`_get_auctions_by_date_modified`)

**Endpoint:** `/api/search/byDateModified/{date}`

**Процес:**
- Виконується пагінація по датах від `date_from` до `date_to`
- Для кожної дати виконується запит з `limit=100`
- Отримані дані парсяться в `AuctionDTO`
- **Попередня фільтрація за датою:** перевіряється, чи `dateCreated` АБО `dateModified` входить у діапазон `[date_from, date_to]`
  - Якщо НІ - аукціон відсівається на цьому етапі
  - Якщо ТАК - додається до списку для подальшої обробки

**Результат:** Список `AuctionDTO` з аукціонами, що пройшли попередню фільтрацію за датою

---

### 1.2. Отримання за датою створення (`_get_auctions_by_date_created`)

**Endpoint:** `/api/search/byDateCreated/{date}` (якщо підтримується)

**Процес:**
- Аналогічно до `_get_auctions_by_date_modified`
- Якщо ендпоінт повертає 404 - метод завершується з порожнім списком
- Дані додаються без попередньої фільтрації (фільтрація буде застосована пізніше)

**Результат:** Список `AuctionDTO` з аукціонами за датою створення

---

## ЕТАП 2: Об'єднання та дедуплікація (`_get_auctions_internal`)

**Процес:**
- Об'єднуються результати з обох методів отримання даних
- Видаляються дублікати за `auction.id`
- Якщо `id` порожній - використовується унікальний ключ `_empty_{index}`

**Результат:** Список унікальних аукціонів без дублікатів

---

## ЕТАП 3: Фільтрація аукціонів (`_should_include_auction`)

Фільтрація виконується в наступному порядку. Якщо аукціон не проходить будь-який фільтр - він відсівається.

### Фільтр 3.1: Перевірка дати (якщо `skip_date_check=False`)

**Умова:** `dateCreated` АБО `dateModified` повинні бути в діапазоні `[date_from, date_to]`

**Код:**
```python
created_in_range = date_from <= date_created <= date_to
modified_in_range = date_from <= date_modified <= date_to
in_date_range = created_in_range or modified_in_range
```

**Результат:** Якщо `in_date_range == False` → аукціон відсівається

---

### Фільтр 3.2: Наявність даних (`auction.data`)

**Умова:** `auction.data` не повинен бути `None` або порожнім

**Результат:** Якщо `auction.data` відсутній → аукціон відсівається

---

### Фільтр 3.3: Статус аукціону

**Умова:** Статус повинен бути активним

**Дозволені статуси:**
- `active`
- `active.tendering`
- `active.auction`
- `active.qualification`
- `active_rectification`
- `active_tendering`
- `active_auction`
- `active_qualification`
- Або будь-який статус, що починається з одного з вищеперелічених

**Код:**
```python
status = auction.data.get('status', '')
is_active = any(
    status.startswith(active_status.replace('_', '.')) or 
    status == active_status or
    status.startswith(active_status.replace('.', '_'))
    for active_status in active_statuses
)
```

**Результат:** Якщо `is_active == False` → аукціон відсівається

---

### Фільтр 3.4: Дата старту торгів

**Умова:** Аукціон не повинен бути завершеним більше ніж 7 днів тому

**Логіка:**
- Якщо `auctionPeriod.startDate` відсутня → аукціон проходить фільтр (дозволяється)
- Якщо `startDate` в майбутньому → аукціон проходить фільтр
- Якщо `startDate` в минулому і з моменту старту пройшло ≤ 7 днів → аукціон проходить фільтр
- Якщо з моменту старту пройшло > 7 днів → аукціон відсівається

**Код:**
```python
auction_start_date = datetime.fromisoformat(auction_period['startDate'])
now = datetime.now(timezone.utc)
days_since_start = (now - auction_start_date).days

if auction_start_date > now:
    pass  # Ще не почався - дозволяється
elif days_since_start <= 7:
    pass  # Завершився недавно - дозволяється
else:
    return False  # Завершився давно - відсівається
```

**Результат:** Якщо аукціон завершився більше 7 днів тому → відсівається

---

### Фільтр 3.5: Класифікаційний код (тип нерухомості)

**Умова:** Аукціон повинен містити код класифікації CAV, що відповідає дозволеним кодам

**Джерело дозволених кодів:** `config/ProZorro_clasification_codes.yaml`

**Дозволені коди (за замовчуванням):**
- `04000000-8` - Нерухоме майно
- `04233000-0` - Адміністративні приміщення в житлових будинках
- `04230000-9` - Адміністративно-офісна нерухомість
- `04130000-8` - Гуртожиток, житловий будинок коридорного типу
- `04232000-3` - Адміністративна будівля
- `70123000-9` - Продаж нерухомості
- `70123200-1` - Продаж нежитлової нерухомості
- `04222000-0` - Магазин, аптека, перукарня тощо
- `06000000-2` - Земельні ділянки
- `04211000-0` - Промислова заводська, фабрична будівля
- `06111000-3` - Землі житлової забудови
- `06112000-0` - Землі громадської забудови
- `05000000-5` - Цілісний майновий комплекс
- `06110000-6` - Земельні ділянки призначені для забудови
- `06128000-5` - Землі промисловості, транспорту тощо

**Процес перевірки:**
1. Перебираються всі `items` з `auction.data.items[]`
2. Для кожного `item` перевіряється `item.classification.scheme == 'CAV'`
3. Перевіряється, чи `item.classification.id` починається з одного з дозволених кодів або точно збігається
4. Якщо знайдено хоча б один відповідний код → аукціон проходить фільтр

**Код:**
```python
items = auction.data.get('items', [])
allowed_classification_codes = self.get_allowed_classification_codes()

is_property_sale = False
for item in items:
    classification = item.get('classification', {})
    if classification.get('scheme') == 'CAV':
        class_id = classification.get('id', '')
        for allowed_code in allowed_classification_codes:
            if class_id.startswith(allowed_code) or class_id == allowed_code:
                is_property_sale = True
                break
```

**Результат:** Якщо `is_property_sale == False` → аукціон відсівається

## ЕТАП 4: Збереження в Excel (`save_auctions_to_csv`)

На цьому етапі всі аукціони, що пройшли фільтрацію на попередніх етапах, зберігаються в Excel файл без додаткової фільтрації за типом операції (продаж/оренда).

---

### Обробка описів через LLM (опціонально)

**Якщо доступний LLM сервіс:**
- Для кожного аукціону виконується парсинг опису через LLM
- Витягуються структуровані дані:
  - Кадастровий номер
  - Площа та одиниця виміру
  - Адреса (область, місто, вулиця, тип вулиці, будинок)
  - Поверх
  - Тип нерухомості
  - Комунальні послуги
  - Інформація про арешти

**Якщо LLM недоступний:**
- Використовуються дані безпосередньо з API

## Статистика фільтрації

Під час виконання збирається статистика:
- `filtered_by_date_count` - відсіяно за датою
- `filtered_by_status_count` - відсіяно за статусом
- `filtered_by_start_date_count` - відсіяно за датою старту торгів
- `filtered_by_property_type_count` - відсіяно за типом нерухомості

Статистика виводиться в консоль після завершення обробки.

---

## Особливості для тижня (7 днів)

Для періоду 7 днів використовується оптимізована обробка:
- Тиждень розбивається на 7 окремих днів
- Кожен день обробляється паралельно в окремому потоці
- Результати зберігаються в окремі тимчасові файли
- Всі файли об'єднуються в один фінальний файл
- Тимчасові файли видаляються після об'єднання
