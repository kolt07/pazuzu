# Індекс відстаней до топонімів (design)

## Мета

Для фільтра **«в радіусі»** (IN_RADIUS) від топоніму потрібно обчислювати відстань не від центру топоніму, а від межі (полігону) топоніму до адрес з точністю до вулиці. Відстань визначається як довжина перпендикуляра до найближчого ребра полігону (або відрізка між двома найближчими вершинами з побудовою висоти до грані топоніму).

## Компоненти

1. **Геокодування топоніму** — отримання меж (bounds/polygon) обраного топоніму (область, місто, район міста).
2. **Граф відстаней** — для кожної адреси з координатами (у `unified_listings.addresses[].coordinates`) обчислюється відстань до кожного топоніму (до межі, не до центру).
3. **Індекс** — окрема колекція (наприклад `toponym_distance_index`), що зберігає:
   - ідентифікатор топоніму (region_id, city_id, city_district_id або slug),
   - ідентифікатор оголошення (source, source_id),
   - відстань у км.
4. **Пошук** — при застосуванні фільтра «в радіусі N км» спочатку шукаємо в індексі список (source, source_id), потім фільтруємо unified_listings за цим списком.

## Поточний стан

- Фільтри **в межах** (INSIDE) та **не в межах** (NOT_INSIDE) реалізовані через regex по полях region/settlement/city_district (domain GeoFilterOperator, CollectionManager._geo_filter_to_mongo).
- **В радіусі** (IN_RADIUS) для координат реалізований через `$geoWithin` + `$centerSphere` (від центру). Для топоніму з геокодуванням — через business GeoFilterService (геокодування адреси → координати → той самий $centerSphere).
- Індекс відстаней до топонімів **не реалізований** — це майбутня оптимізація. Поки що IN_RADIUS від топоніму можна реалізувати: геокодувати топонім → взяти центр bounds → використати існуючу логіку радіусу від точки.

## Файли

- `domain/models/filter_models.py` — GeoFilterOperator (INSIDE, NOT_INSIDE, IN_RADIUS).
- `domain/managers/collection_manager.py` — _geo_filter_to_mongo (INSIDE/NOT_INSIDE/IN_RADIUS).
- `config/search_fields.yaml` — секція geo, оператори inside, not_inside, in_radius.
