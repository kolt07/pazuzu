# Інтеграція LangChain

Цей документ описує безпечну інтеграцію LangChain у проєкт з дотриманням архітектурних принципів.

## Архітектурні принципи

### Основні правила

1. **LLM НІКОЛИ не мають прямого доступу до баз даних, API або файлової системи**
   - Всі операції з даними виконуються ВИКЛЮЧНО через MCP-сервери (tools)
   - Агент не може генерувати прямі MongoDB, SQL або HTTP-запити

2. **УСІ операції з даними виконуються ВИКЛЮЧНО через MCP-сервери (tools)**
   - MCP-сервери визначають жорсткі контракти для:
     - інспекції схем даних
     - аналітики та агрегацій
     - реєстру джерел даних
     - генерації звітів

3. **LangChain використовується ЛИШЕ як runtime для агентів та оркестрації**
   - Бізнес-логіка, валідація, безпека та обмеження реалізуються на сервері
   - Агент не може змінювати дані без явного дозволу

4. **Явний контроль замість автономії**
   - Використовується явний цикл агента (plan → act → observe)
   - Максимальна кількість ітерацій обмежена
   - Всі дії логуються

## Компоненти

### LangChainAgentService

Сервіс `business/services/langchain_agent_service.py` реалізує безпечну інтеграцію LangChain.

#### Основні можливості

- **Явний цикл агента**: plan → act → observe з максимальною кількістю ітерацій
- **MCP Tools**: Всі операції з даними через LangChain Tools, які викликають MCP-сервери
- **Валідація**: Перевірка запитів на рівні сервісу
- **Безпека**: Перевірка на заборонені оператори ($regex тощо)
- **Логування**: Детальне логування всіх дій агента

#### Використання

```python
from business.services.langchain_agent_service import LangChainAgentService
from config.settings import Settings

settings = Settings()
agent_service = LangChainAgentService(settings)

# Обробка запиту
response = agent_service.process_query(
    user_query="Скільки аукціонів у Львівській області?",
    stream_callback=lambda text: print(text)  # Опціонально
)
```

## Доступні Tools

### Schema MCP Tools

#### `get_database_schema`
Отримує повну схему метаданих всіх колекцій бази даних.

**Використання**: Для дослідження структури БД перед виконанням запитів.

#### `get_collection_info`
Отримує детальну інформацію про конкретну колекцію.

**Параметри**:
- `collection_name` (string): Назва колекції (`prozorro_auctions` або `llm_cache`)

#### `get_data_dictionary`
Отримує повний Data Dictionary з описом всіх колекцій та полів.

### Query Builder MCP Tools

#### `execute_query`
Виконує безпечний запит до MongoDB.

**ВАЖЛИВО**: 
- `$regex` оператор ЗАБОРОНЕНИЙ
- Для фільтрації за статусом або регіоном/містом використовуй `analytics-mcp`

**Параметри**:
- `query` (dict): Абстрактний запит у форматі:
  ```json
  {
    "collection": "prozorro_auctions",
    "filters": {...},
    "join": [...],
    "projection": [...],
    "limit": 10
  }
  ```

#### `get_allowed_collections`
Отримує список дозволених колекцій для запитів.

### Analytics MCP Tools

#### `execute_analytics`
Виконує аналітичний запит з метриками та агрегаціями.

**Параметри**:
- `query` (dict): Аналітичний запит:
  ```json
  {
    "metric": "average_price_per_m2",
    "groupBy": ["region"],
    "filters": {
      "status": "active",
      "region": "Львівська"
    }
  }
  ```

#### `list_metrics`
Отримує список доступних метрик для аналітики.

### Report MCP Tools

#### `generate_report`
Генерує звіт у вказаному форматі (xlsx, csv, json).

**Параметри**:
- `request` (dict): Запит на генерацію звіту:
  ```json
  {
    "format": "xlsx",
    "template": "auction_summary",
    "dataSource": "analytics-mcp:{\"metric\":\"average_price_per_m2\"}",
    "columns": ["region", "avg_price_m2"]
  }
  ```
- `return_base64` (bool): Чи повертати файл у base64 (за замовчуванням: true)

#### `list_templates`
Отримує список доступних шаблонів звітів.

## Поведінка агента

Агент дотримується такого алгоритму:

1. **Аналіз запиту**: Агент аналізує запит користувача
2. **Визначення інформації**: Визначає, якої інформації бракує
3. **Виклик MCP-інструментів**: Викликає MCP-інструменти для інспекції схем і можливостей
4. **Побудова плану**: Будує план, використовуючи ЛИШЕ доступні інструменти
5. **Виконання**: Виконує інструменти покроково
6. **Фінальна відповідь**: Використовує у фінальній відповіді ТІЛЬКИ результати MCP

### Логування

Агент логує:
- Чому був викликаний кожен MCP tool
- Які припущення були перевірені
- Які обмеження має результат

## Безпека

### Валідація на рівні сервісу

Сервіс перевіряє запити на:
- Порожні запити
- Спроби прямого доступу до БД (паттерни типу `db.`, `collection.`, `mongodb://`)
- Заборонені операції (`insert`, `update`, `delete`, `drop`)

### Валідація на рівні tools

Tools перевіряють:
- Заборонені оператори (`$regex`, `$where`, `$eval`)
- Дозволені колекції
- Максимальну кількість результатів
- Валідність параметрів

## Налаштування

### Провайдери LLM

Сервіс підтримує:
- Google Gemini (через `langchain-google-genai`)
- OpenAI (через `langchain-openai`)
- Anthropic Claude (через `langchain-anthropic`)

Налаштування в `config/settings.py`:
```python
llm_provider = "gemini"  # або "openai", "anthropic"
llm_model_name = "gemini-2.5-flash"
llm_api_keys = {
    "gemini": "your-api-key",
    "openai": "your-api-key",
    "anthropic": "your-api-key"
}
```

### Обмеження

- Максимальна кількість ітерацій: 10
- Максимальна кількість викликів tools за ітерацію: 5
- Максимальна кількість результатів запиту: 100

## Порівняння з llm_agent_service

| Аспект | llm_agent_service | langchain_agent_service |
|--------|-------------------|-------------------------|
| Runtime | Gemini API напряму | LangChain |
| Цикл агента | Неявний (через Gemini function calling) | Явний (plan → act → observe) |
| Логування | Базове | Детальне з поясненнями |
| Валідація | На рівні tools | На рівні сервісу + tools |
| Контроль | Обмежений | Повний контроль |

## Рекомендації

1. **Використовуйте LangChain для продакшену**: Більший контроль, краще логування, явний цикл
2. **Використовуйте llm_agent_service для прототипування**: Швидше налаштування, менше залежностей
3. **Завжди вмикайте логування**: Для діагностики та моніторингу
4. **Обмежуйте кількість ітерацій**: Для запобігання нескінченним циклам
5. **Валідуйте запити**: На рівні сервісу та tools

### Запит з трансляцією

```python

    print(text, end='', flush=True)

    "Знайди топ-10 аукціонів за ціною за м² у Львівській області",
    stream_callback=stream_callback
)
```

## Troubleshooting

### Помилка: "Tool не знайдено"
Перевірте, чи всі tools правильно зареєстровані в `_create_tools()`.

### Помилка: "Досягнуто максимум ітерацій"
Агент не може завершити запит за 10 ітерацій. Спробуйте:
- Спростити запит
- Збільшити `MAX_ITERATIONS` (не рекомендується)
- Перевірити, чи всі необхідні tools доступні

### Помилка: "$regex оператор заборонений"
Використовуйте `analytics-mcp` для фільтрації за статусом або регіоном.
