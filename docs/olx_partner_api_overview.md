# Огляд OLX Partner API

Короткий огляд можливостей OLX Partner API та варіантів використання в застосунку Pazuzu як додаткового джерела даних до ProZorro.

## Джерело

- Специфікація: `docs/partner_api.yaml` (OpenAPI 3.0.2)
- Базовий URL: `https://www.olx.ua/api/partner`
- Документація однакова для всіх європейських сайтів OLX (в т.ч. olx.ua).

---

## Аутентифікація

- **OAuth 2.0**. Обов’язковий заголовок `Version` (наприклад `Version: 2.0`).
- **Типи grant:**
  - `client_credentials` — доступ від імені клієнта (конфіг: категорії, міста, регіони тощо). Не можна публікувати оголошення.
  - `authorization_code` — від імені користувача: публікація/редагування оголошень, повідомлення.
  - `refresh_token` — оновлення access token.
- **Scopes:** `v2`, `read`, `write` (список обмежений налаштуваннями партнера).
- Потрібні `client_id` та `client_secret` від OLX (отримуються після схвалення заявки).

---

## Основні групи ендпоінтів

| Група | Ендпоінти | Призначення |
|-------|-----------|-------------|
| **Користувачі** | `GET /users/me`, `GET /users/{id}` | Поточний користувач, профіль іншого користувача |
| **Геолокація** | `/regions`, `/regions/{id}`, `/cities`, `/cities/{id}`, `/cities/{id}/districts`, `/districts`, `/locations` | Регіони, міста, райони — довідники для оголошень |
| **Довідники** | `/languages`, `/currencies` | Мови та валюти (мультимовність) |
| **Категорії** | `/categories`, `/categories/{id}`, `/categories/{id}/attributes`, `/categories/suggestion?q=...` | Дерево категорій, атрибути категорії, підказка категорії за текстом (наприклад заголовком) |
| **Оголошення (adverts)** | `GET/POST /adverts`, `GET/PUT/DELETE /adverts/{id}`, `/adverts/{id}/commands`, `/adverts/{id}/statistics`, `/adverts/{id}/logos`, `/adverts/{id}/packets` | Створення, оновлення, видалення, активація/деактивація, статистика, пакети |
| **Повідомлення** | `/threads`, `/threads/{id}`, `/threads/{id}/messages`, `/threads/{id}/messages/{id}`, `/threads/{id}/commands` | Листування з користувачами OLX щодо оголошень |
| **Пакети та білінг** | `/packets`, `/zones`, `/users/me/packets`, `/users/me/billing` | Платні пакети оголошень, зони, рахунок |

## Пошук і отримання оголошень за фільтрами (загальнодоступна документація)

### Висновок з офіційних джерел

За **загальнодоступною документацією OLX** (Partner API, OLX Group Developer Hub, регіональні портали, зокрема developer.olx.ua):

- **Публічного API для пошуку оголошень за фільтрами немає.**  
  Немає ендпоінтів типу «пошук по категорії / регіону / ціні / даті» по всьому каталогу OLX. Усі офіційні API орієнтовані на **керування оголошеннями авторизованого користувача** (публікація, редагування, активація, отримання списку своїх оголошень тощо).

- **Єдине отримання списку оголошень з параметрами** — це **GET /adverts** у Partner API (`docs/partner_api.yaml`). Цей ендпоінт повертає **лише оголошення поточного користувача** і підтримує такі параметри:
  - `offset` — зміщення для пагінації;
  - `limit` — кількість записів на сторінку;
  - `external_id` — фільтр за зовнішнім ID (наприклад з вашої системи);
  - `category_ids` — список ID категорій (через кому).

  Фільтрів за ціною, датою публікації, регіоном/містом, текстом пошуку або статусом в офіційній специфікації **немає**. Це саме фільтрація **власних** оголошень для партнерського клієнта.

- **Інші офіційні продукти** (OLX RE API, Advert API на developer.olxgroup.com, Listings API на api-documentation.olx.ba) також не надають публічного пошуку: лише створення/редагування/отримання одного оголошення за ID тощо. На developer.olx.ua явно зазначено, що через API можна **керувати лише власними оголошеннями**.

### Джерела

- **Partner API (olx.ua):** `docs/partner_api.yaml` — GET /adverts (параметри offset, limit, external_id, category_ids).
- **Портал розробників OLX Україна:** https://developer.olx.ua/api/doc (TLS 1.2+, обмеження по запитах, керування власними оголошеннями).
- **OLX Group Developer Hub:** https://developer.olxgroup.com/ — Advert API, Real Estate API тощо; орієнтація на публікацію та керування оголошеннями, без публічного пошуку.
- **OLX Bosnia Listings API:** https://api-documentation.olx.ba/listings — GET /listings/:id (одне оголошення за ID), без ендпоінту пошуку/списку за фільтрами.

Для отримання **актуальних оголошень за фільтрами по всьому сайту OLX** офіційний API можливостей не надає; питання доступу до таких даних потребує звернення безпосередньо до OLX.

---

## Важливе обмеження щодо даних

- **GET /adverts** повертає лише оголошення поточного авторизованого користувача (з параметрами `offset`, `limit`, `external_id`, `category_ids`).
- Публічного пошуку або вибірки всіх оголошень OLX за категорією/датою/регіоном/ціною в офіційному API **немає**.

Тобто OLX Partner API **не є джерелом ринкових даних** у тому ж сенсі, що ProZorro: через нього неможливо отримувати масові списки чужих оголошень для аналітики або моніторингу ринку.

---

## Як можна задіяти API в Pazuzu (разом із ProZorro)

1. **Публікація на OLX після перемоги на ProZorro**  
   Користувач виграв лот на ProZorro.Sale (наприклад нерухомість/активи) → застосунок може допомогти створити оголошення на OLX через `POST /adverts` (потрібна OAuth-авторизація користувача в OLX). Поля `external_id` та `external_url` дозволяють прив’язати оголошення до тендера/аукціону в ProZorro.

2. **Довідники для нормалізації та UI**  
   Категорії, регіони, міста, валюти, мови — можна використовувати для:
   - маппінгу локацій/типів об’єктів між ProZorro та OLX;
   - підказки категорії за назвою лоту (`/categories/suggestion?q=...`);
   - узгодження валют і мов при експорті даних або формуванні оголошень.

3. **Управління власними оголошеннями**  
   Для користувачів, які ведуть оголошення через Pazuzu: перегляд списку, оновлення, активація/деактивація, перегляд статистики, робота з повідомленнями (threads/messages).

4. **Що не покриває API**  
   Неможливо використовувати OLX як додаткове джерело «каталогу ринкових пропозицій» на кшталт ProZorro: масове читання чужих оголошень для порівняння цін або аналітики через цей API недоступне.

---

## Висновок

- **ProZorro** у Pazuzu — джерело даних про закупівлі/аукціони (читання, фільтрація, збереження, звіти).
- **OLX Partner API** — інструмент для публікації та керування **власними** оголошеннями на OLX та отримання довідникових даних. Його доцільно використовувати як доповнення до ProZorro (наприклад сценарій «виграв на ProZorro → опублікував на OLX»), а не як альтернативне джерело ринкових оголошень.
