# Pazuzu — стек технологій та оцінка складності

Документ призначений для використання в резюме: опис стеку, реалізованих фіч та рівня проектування.

---

## 1. Стек технологій

### Backend
- **Python 3.x** — основний стек
- **FastAPI** — REST API для Telegram Mini App
- **Uvicorn** — ASGI-сервер
- **MongoDB (PyMongo)** — основне сховище даних
- **APScheduler** — планування регламентних задач (оновлення даних, звіти, нагадування)

### LLM та агентна архітектура
- **LangChain** — оркестрація агентів, інструменти, пам'ять (buffer, summary, vector)
- **LangGraph** — опційний state-graph для циклу агент→tools→агент
- **MCP (Model Context Protocol)** — окремі процеси-сервери з інструментами для агентів
- **Провайдери LLM**: Google GenAI (Gemini), OpenAI, Anthropic, Ollama — з fallback та rate limiting

### Джерела даних та парсинг
- **ProZorro / ProZorro.Sale API** — аукціони нерухомості
- **OLX** — скрапінг (BeautifulSoup, lxml, Selectolax, Parsel), категорії нерухомості та землі
- **Google Maps API** — геокодування адрес, POI (для гео-аналізу приміщень)
- **Mapbox Vector Tiles, Shapely** — робота з кадастровими та геоданими

### Клієнт та доставка
- **python-telegram-bot** — Telegram Bot API (меню, кнопки, документи, WebApp)
- **Telegram Mini App** — веб-інтерфейс у WebView (FastAPI + статика), валідація initData (HMAC-SHA256)

### Дані та аналітика
- **Pandas, OpenPyXL** — обробка та експорт (XLSX)
- **YAML** — конфігурація (config, prompts, data dictionary, словники топонімів/типів)

### Інфраструктура та якість
- **Міграції БД** — власний скриптовий фреймворк (40+ міграцій)
- **Логування** — структуроване, кореляційні ID, придушення сторонніх логів
- **Конфіг** — YAML + env, міграція конфігу при старті

## 2. Архітектура та рівень проектування

### Шарова архітектура
- **Domain layer** — сутності (UnifiedListing, ProzorroAuction, RealEstateObject тощо), колекції з логікою filter/sort/export, Gateway, CollectionManager
- **Data layer** — репозиторії MongoDB, індекси та data_dictionary (єдине джерело правди для полів/індексів)
- **Business layer** — сервіси (LLM, геокодування, пайплайни, експорт, аналітика, кеші), без прямого доступу агентів до БД
- **Transport** — Telegram Bot, Mini App API, MCP tools як контракт для агентів

### Мультиагентна система
- **SecurityAgent** — перевірка запиту на шкідливі патерни, rate limit, сповіщення адмінів
- **InterpreterAgent** — rule-based fast path + LLM fallback, структурований намір (intent, collections, filters, period)
- **PlannerAgent** — побудова пайплайну кроків (оновлення даних → вибірка → експорт)
- **AnalystAgent** — виконання запитів/агрегацій через MCP, тимчасові вибірки, експорт у файли
- **QueryStructureAgent** — структура запиту (фільтри, метрики, гео) з природної мови
- **PipelineBuilderAgent** — генерація пайплайнів з контексту схеми та кешів
- **AssistantAgent** — координація, контекст бесіди, виклик під-агентів
- **LangChain Tool Agent** — один агент з повним набором MCP-інструментів та пам’яттю для складних/розмовних запитів

### MCP як domain API
- 7 окремих MCP-серверів: schema, query_builder, analytics, report, export, geocoding, data_update
- Агент не генерує Mongo-поля — лише логічні поля; PipelineInterpreter + FieldMappingService маплять на фізичну схему
- Валідація запитів до виконання, заборона $where/$regex тощо

### Якість коду та процеси
- Глосарій розробника, історія розробки, Cursor rules (ProZorro, промпти, price metrics, LLM-архітектура, domain layer)
- Єдиний стиль промптів (завдання → контекст → закріплення формату)
- Міграції при зміні структури БД, оновлення domain/data_dictionary при зміні колекцій

## 3. Складність реалізованих фіч

| Категорія | Фіча | Складність | Коментар |
|-----------|------|------------|----------|
| **Дані** | Збір аукціонів ProZorro.Sale (search API, нормалізація) | Середня | API, фільтри CPV, статуси, збереження |
| **Дані** | Скрапінг OLX (нежитлова нерухомість, земля за типами/регіонами) | Висока | Багато категорій, Phase 1 (сирі) + Phase 2 (LLM одним потоком), канонічні URL, дедуплікація |
| **Дані** | Зведена колекція unified_listings (ProZorro + OLX) | Висока | Нормалізація полів, root geo з addresses, price_metrics (price_per_m2/price_per_ha за правилами), real_estate_object_refs |
| **Дані** | Геокодування, кеш, root geo, city_district (райони міста) | Висока | Google Maps, fallback по топонімах, заповнення root geo з масиву адрес (majority/пріоритет) |
| **Дані** | Аналітичні витяги (analytics_extracts), агрегації по метриках/містах/районах | Середня–висока | Окрема колекція, доменний сервіс, формули метрик, MCP tools |
| **Дані** | Price analytics (зведені метрики по періодах/регіонах), індикатори ціни (IQR) | Середня | Агрегати по днях/тижнях/місяцях, показники «вигідна»/«аномальна» ціна |
| **LLM** | Парсинг описів оголошень (адреси, площі, теги, ОНМ) | Середня | Структурований вивід, кеш LLM, retry при 429 |
| **LLM** | Мультиагентний пайплайн (безпека → інтерпретатор → планувальник → виконання) | Висока | Двоступенева маршрутизація (rule-based + LLM), кеш наміру, валідація кроків плану |
| **LLM** | LangChain-агент з MCP tools, пам’ять (buffer/summary/vector), error recovery | Висока | ReAct-стиль підказок, iteration limit, thinking budget, опційно LangGraph |
| **LLM** | QueryStructureAgent, PipelineBuilderAgent (структура запиту з природної мови) | Висока | Нормалізація топонімів, відносні дати (RelativeDateResolver), логічні поля без Mongo-шляхів |
| **Експорт** | Експорт за ідентифікаторами, тимчасові вибірки (batch_id), артефакти з TTL | Середня | Ліміти по рядках/файлу/користувачу на день, відправка файлу в Telegram після tool call |
| **Звіти** | Шаблони звітів, генерація за параметрами, регламентне формування | Середня | Колекція шаблонів, scheduled_report у планувальнику, доставка у Telegram |
| **Планувальник** | Події: оновлення даних, звіти, нагадування (cron/once), scope system/user | Середня | APScheduler, колекція scheduled_events, тільки адміни — data_update |
| **Клієнт** | Telegram Bot (меню, кнопки, документи, адмін-дії, тестування агента) | Середня | Ролі, initData для Mini App |
| **Клієнт** | Telegram Mini App (пошук, чат з LLM, звіти, адмін, валідація initData) | Середня | FastAPI, статика, HMAC перевірка, дубль функцій бота |
| **Безпека** | Security layer: патерни, rate limit, max complexity, сповіщення адмінів | Середня | Guardrails до обробки запиту |
| **Операційне** | Міграції БД (40+), data_dictionary, оновлення domain layer при зміні схеми | Висока | Версійність міграцій, чеклисти в Cursor rules |
| **Тестування** | Тест-агент (генерація кейсів, порівняння з БД, звіт по кейсах) | Середня | Прямий доступ до БД лише для перевірки очікувань |

## 4. Підсумкова оцінка для резюме

**Рівень складності застосунку:** високий  
- Багато джерел даних (ProZorro, OLX), зведення в єдину модель з гео та ціновими метриками  
- Мультиагентна LLM-система з чітким поділом ролей (інтерпретатор, планувальник, аналітик, помічник), MCP як єдиний інтерфейс до даних  
- Domain-driven підхід: сутності, колекції, Gateway, маппінг логічних полів на MongoDB  
- Повний цикл: збір → нормалізація → аналітика → експорт/звіти та доставка через Telegram/Mini App  

**Рівень проектування:** високий  
- Чіткі правила архітектури (LLM не виконує дії, не генерує Mongo-поля, мінімальний доступ агентів)  
- Документовані доменні правила (ProZorro, price metrics, промпти), глосарій, історія змін  
- Систематичні міграції та оновлення domain/config при зміні структури  
- Rule-based маршрутизація перед LLM, confidence-based fallback, security layer  

**Короткий опис для резюме (1–2 речення):**  
«Backend-застосунок на Python: збір та нормалізація даних з ProZorro.Sale та OLX, зведена колекція нерухомості з геокодуванням і ціновими метриками. Мультиагентна LLM-система (LangChain, MCP) з доменним шаром, Telegram Bot та Mini App (FastAPI), регламентні звіти та експорт.»

**Розширений опис для резюме (пункти):**  
- Python, FastAPI, MongoDB, LangChain/LangGraph, MCP, Telegram Bot API та Mini App  
- Збір даних: ProZorro.Sale API, скрапінг OLX (категорії нерухомості/землі), геокодування (Google Maps)  
- Domain layer: сутності, колекції, маппінг полів; зведена колекція unified_listings з root geo та price metrics  
- Мультиагентна архітектура: інтерпретатор (rule-based + LLM), планувальник, аналітик, security layer; інструменти через MCP (schema, query, analytics, export, geocoding, data_update)  
- Клієнт: Telegram Bot (меню, файли, адмін), Mini App на FastAPI з валідацією initData  
- Регламентні задачі (APScheduler), міграції БД (40+), конфіг/промпти в YAML, глосарій та правила архітектури
