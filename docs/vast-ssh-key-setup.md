# SSH-ключ для нової машини та реєстрація в Vast.ai

Коротка інструкція: згенерувати пару ключів на **цьому** комп’ютері, додати **публічний** ключ у акаунт Vast, після чого підключатися до інстансів лише з відповідним **приватним** ключем.

Офіційні джерела: [SSH Connection](https://docs.vast.ai/documentation/instances/connect/ssh), [Keys](https://docs.vast.ai/documentation/reference/keys).

## 1. Згенерувати нову пару ключів

На **Windows** у PowerShell (рекомендований тип ключа — `ed25519`):

```powershell
ssh-keygen -t ed25519 -C "vast-your-pc-name"
```

- `-C "..."` — необов’язковий коментар для ідентифікації ключа (наприклад, ім’я ПК); на безпеку не впливає.
- Якщо залишити шлях за замовчуванням, з’являться два файли у `%USERPROFILE%\.ssh\`:
  - `id_ed25519` — **приватний** ключ (не копіювати в чати, не викладати в репозиторій).
  - `id_ed25519.pub` — **публічний** ключ (його і додають у Vast).

Якщо на цій машині вже є `id_ed25519` і ви не хочете його перезаписувати, задайте інше ім’я файлу під час генерації (наприклад, `id_ed25519_vast`).

На **Linux / macOS** команда така сама; типові шляхи — `~/.ssh/id_ed25519` та `~/.ssh/id_ed25519.pub`.

## 2. Скопіювати публічний ключ

**PowerShell:**

```powershell
Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub
```

Скопіюйте весь рядок, що починається з `ssh-ed25519` (один рядок).

**Linux / macOS:**

```bash
cat ~/.ssh/id_ed25519.pub
```

## 3. Зареєструвати ключ у Vast.ai

1. Увійдіть у [консоль Vast](https://cloud.vast.ai/).
2. Відкрийте сторінку керування ключами: [https://cloud.vast.ai/manage-keys/](https://cloud.vast.ai/manage-keys/).
3. У розділі **SSH Keys** натисніть **+ New**.
4. Вставте вміст `*.pub` у поле та збережіть.

Після збереження ключ з’явиться у списку і буде використовуватися для **майбутніх** інстансів, створених уже після додавання ключа.

## 4. Що важливо знати

- **Існуючі інстанси** новий акаунтний ключ **автоматично не отримують**. Для вже запущеного інстансу потрібно додати доступ через інтерфейс цього інстансу (розділ SSH) або інструменти Vast (див. [документацію SSH](https://docs.vast.ai/documentation/instances/connect/ssh)).
- **Віртуальні машини (VM)** на Vast: ключі краще мати в акаунті **до** створення VM; зміна ключів для VM часто вимагає пересоздання машини — деталі в офіційній документації.
- На інстансі ввімкнена лише аутентифікація за ключем; **вхід за паролем SSH вимкнено**.

## 5. Підключення до інстансу

Після створення інстансу відкрийте в консолі Vast іконку **SSH** і скопіюйте готову команду (зазвичай це `ssh` з нестандартним портом, користувач часто `root`).

Якщо приватний ключ не за ім’ям за замовчуванням, вкажіть його явно, наприклад:

```powershell
ssh -i $env:USERPROFILE\.ssh\id_ed25519 -p <PORT> root@<IP>
```

## 6. Якщо не пускає: `Permission denied (publickey)`

1. Переконайтеся, що **саме цей** публічний ключ збережено на [сторінці Keys](https://cloud.vast.ai/manage-keys/).
2. Переконайтеся, що ви підключаєтеся **тим приватним ключем**, який відповідає цьому `.pub`.
3. На Linux/macOS перевірте права на приватний ключ: `chmod 600 ~/.ssh/id_ed25519`.
4. Запустіть з діагностикою: `ssh -vv -p <PORT> root@<IP>` і перегляньте, який ключ пропонується клієнту.

## 7. Альтернатива: Vast CLI

Можна встановити CLI і створити ключ через Vast (потрібен API key з [сторінки CLI](https://cloud.vast.ai/cli/)):

```powershell
py -m pip install vastai
vastai set api-key YOUR_API_KEY
vastai create ssh-key
```

Деталі — у розділі **Vast CLI** на сторінці [SSH Connection](https://docs.vast.ai/documentation/instances/connect/ssh).
